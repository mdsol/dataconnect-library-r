## Security Policy

### Reporting a Vulnerability

If you discover a security vulnerability in this package, please report it to us privately.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to: helpdesk@mdsol.com

Please include the following information:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We will acknowledge your email within 48 hours and provide a more detailed response within 7 days.

### Supported Versions
| Version | Supported          |
| ------- | ------------------ |
| 1.0.0   | ✅                |
| 1.0.1   | ✅                |

### Security Update Process

1. Security issues are prioritized and addressed immediately
2. Fixes are released as patch versions
3. Security advisories are published on GitHub
4. Users are notified through release notes and package updates
